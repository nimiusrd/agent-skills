# リファクタリング候補テンプレート

このテンプレートは、リファクタリング候補を `refactoring-discoveries/` ディレクトリに MD ファイルとして記録する際のフォーマットを定義する。

## ファイル命名規則

```
refactoring-discoveries/{NNN}-{function_name}.md
```

- `{NNN}`: 3桁の連番（001, 002, ...）
- `{function_name}`: 対象の関数/メソッド名（スネークケース）
- 例: `refactoring-discoveries/001-parse_user.md`

## テンプレート

```markdown
# リファクタリング候補: {NNN}-{function_name}

## 対象
- **ファイル**: `{file_path}`
- **行範囲**: L{start}-L{end}
- **関数/メソッド名**: `{function_name}`
- **スコープレベル**: Level {1|2|3}

## 根拠

### 静的指標
- **循環複雑度**: {値または推定}
- **行数**: {値}
- **重複**: {重複箇所の説明、なければ「なし」}
- **結合度**: {依存先の数、結合の種類}

### 動的指標
- **churn（変更頻度）**: {git log から算出した変更回数/期間}
- **バグ修正頻度**: {fix/bug を含むコミットの回数}

## 期待効果
- {可読性向上 / 複雑性低減 / 重複排除 / 責務明確化 など}

## 影響範囲
- **呼び出し元**: {関数/ファイルのリスト}
- **依存先**: {関数/ファイルのリスト}

## Gate B 材料
- **テスト状況**: {カバー済み / 未カバー}
- **カバー済みパス**: {正常系 / 境界値 / 異常系 / 副作用}
- **未カバーパス**: {不足しているテストの具体}

## 推奨アプローチ
1. {具体的なリファクタリング手順のステップ1}
2. {ステップ2}
3. {ステップ3}

## メタデータ
- **発見日時**: {YYYY-MM-DD}
- **ステータス**: 未着手
- **分析観点**: {複雑性分析 / 重複分析 / 依存分析}
```

## ステータスの遷移

| ステータス | 意味 |
|-----------|------|
| 未着手 | 発見済み、未実施 |
| 実行中 | リファクタリング作業中 |
| 完了 | リファクタリング完了・検証済み |
| 中止 | Gate 未達等で中止 |
